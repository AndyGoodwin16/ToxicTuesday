What is League of Legends?

"League of Legends (League or LoL) is a multiplayer online battle arena video game. Two teams of five players battle in player-versus-player combat, each team occupying and defending their half of the map. Each of the ten players controls a character, known as a "champion", with unique abilities and differing styles of play. During a match, champions become more powerful by collecting experience points, earning gold, and purchasing items to defeat the opposing team. In League's main mode, Summoner's Rift, a team wins by pushing through to the enemy base and destroying their "Nexus", a large structure located within.”

What is the motivation for this tool?

My friends from college and I have been playing League since 2013 – it is one of our favorite methods of staying in touch and hanging out since many of us have now moved around the country. Regularly, we will play “in-houses”, in which ten of us are split into two teams of five to play against one another for the ultimate bragging rights. Due to privacy concerns, Riot (development company of League) has made these private, custom matches only viewable by the participants and thus, the data is inaccessible. Therefore, we requested and were granted access to generate our own tournament codes for these custom matches, which allows for Riot to store the game data and for us to access it via their API. The Toxic Tuesday website is an amalgamation of certain game data from all the matches we have played under the tournament code system.

What is the general work flow?

All game data from custom matches created via a tournament code are accessible through Riot’s API given a valid Riot API key and the game ID of the desired match. The game data is returned in a JSON format, so using a Python script, the desired data was gathered into several different table formats and stored in a SQlite file for future use. Another Python script imported, transformed, and exported the stored data as JSON files in the desired format. A JavaScript file, using Plotly and DataTables, imported the JSON files and generated interactable charts/tables. Finally, HTML files displayed data/charts/tables on an interactive website hosted via GitHub Pages.

What future data and/or visualizations will be included?

There are three different JSON files accessible per game ID – game data, pick and ban data, and timeline data. The tool in its current form only accesses the first two files to generate the displayed data. Timeline data contains much more specific data to the individual matches, like specific locations and statistics of certain characters at certain points during the game. Valuable metrics about playstyle of specific participants may be gleamed from this data and thus, it will certainly be explored in the near future.

Can this tool be applied to any group looking to track custom game data?

Yes, anyone with access from Riot to generate their own tournament codes and API keys (both of which Riot gladly gives out) can modify this tool for their own use. While there are some specific lines of code targeted to our specific group and other special case scenarios, there is nothing too difficult here that it cannot be generalized. Anyone who wishes to use this tool for their own purposes, please contact me and I would be happy to help with setup!
